2.3. Database Modeling

üîÑ How to Check if Database Data is Outdated

Key Strategy: Add timestamp fields to track when data was last updated, then compare against freshness requirements.

üìä Do I Need to Store Massive Data?

Short Answer: NO! Your database can be small and smart:

  • Store selectively: Only cache frequently accessed data (current season stats, popular players)
  • Use hybrid approach: Database for fast access + API for comprehensive data
  • Smart caching: Store processed/calculated data (trends, comparisons) rather than raw historical data
  • Typical size: A well-designed NBA database might only need 100MB-1GB for core functionality

Example: Store current roster (450 players √ó 20 stats) + recent games = very manageable size!

üîÑ Hybrid Data Strategy: When to Use Database vs API

You've got it right! Here's the smart approach:

  • Database (Fast Cache): Popular players, current season, recent games, trending stats
  • API (On-Demand): Historical players, old games, rare searches, detailed archives

Example Flow:

  • ‚úÖ User searches "LeBron James" ‚Üí Database (popular, fast response)
  • ‚úÖ User searches "1990s bench player stats" ‚Üí API (rare, okay to be slower)
  • ‚úÖ User views today's games ‚Üí Database (fresh, frequently accessed)
  • ‚úÖ User explores 1985 playoffs ‚Üí API (historical, infrequent)

Smart Benefits: 95% of queries are lightning fast, 5% take a bit longer but still work perfectly!

  • Design database schema using SQLAlchemy ORM models
  • Add timestamp fields: created_at, updated_at, last_api_sync
  • Implement data freshness rules (player stats: 1 hour, scores: 5 minutes)
  • Create "is_stale()" function to check if data needs refresh
  • Design efficient indexes and relationships
  • Set up Alembic for database migrations
  • Define data caching strategy (local vs. live API)
